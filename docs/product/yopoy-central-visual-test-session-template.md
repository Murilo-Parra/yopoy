## Regras anti-refatoração

- feedback de um participante não vira implementação imediata;
- não reabrir autenticação, cadastro ou tenant provisório;
- não reabrir migration;
- não trazer fiscal real para o MVP;
- não transformar pedido isolado em mudança estrutural;
- agrupar padrões entre sessões antes de priorizar;
- classificar o feedback no backlog antes de propor qualquer alteração;
- exigir etapa própria e allowlist explícita para toda mudança futura.
